# 聊天功能概览

Open WebUI 提供了一整套面向对话场景的能力，用于增强你与 AI 模型的交互体验。本页汇总了最重要的聊天功能，并附上深入说明页面的链接。

## 核心聊天功能

- **[📁 文件夹与项目](./conversation-organization.md)**：把文件夹转化为强大的项目工作区，并绑定自定义系统提示词与知识库。

- **[🔗 URL 参数](./url-params.md)**：通过 URL 参数配置聊天会话，快速预设模型、工具和其他能力。

- **[⚙️ 聊天参数](./chat-params.md)**：在会话级、账户级或模型级控制系统提示词和高级参数。

- **[✨ 自动补全](./autocomplete.md)**：由 AI 驱动的文本预测能力，可借助任务模型更快撰写提示词。

- **[🗨️ 聊天分享](./chatshare.md)**：可在本地或通过 Open WebUI Community 平台分享对话，并控制隐私设置。

- **[🔍 历史与搜索](./history-search.mdx)**：浏览并搜索以往对话，或让模型通过原生工具自主检索历史记录。

- **[🕒 时间感知](./temporal-awareness.mdx)**：介绍模型如何理解时间与日期，以及用于精确时间计算的原生工具。

- **🎙️ 语音模式**：使用语音模式按钮即可与模型进行免提语音对话。该层包含静音/取消静音切换开关，可在不结束会话的情况下暂时关闭麦克风——在嘈杂或公共环境中非常实用。

- **[🧠 推理与 Thinking 模型](./reasoning-models.mdx)**：为使用 thinking 标签生成内部思维链的模型提供专门支持。

- **[💬 跟进问题](./follow-up-prompts.md)**：在模型回复后自动生成推荐的后续提问。

- **[⏱️ 自动化](./automations.mdx)**：按小时、天、周、月或自定义 RRULE 自动执行提示词，并管理周期性运行。

- **[✅ 任务管理](./task-management.mdx)**：让 Agent 型模型在聊天中维护结构化任务/待办列表，以支持多步骤工作流和透明进度跟踪。

- **技能提及**：在聊天输入框中使用 `$` 即可即时提及并激活 [技能](/features/workspace/skills)，把其 manifest 注入当前对话。

- **写作与内容块**：当模型回复中包含冒号围栏块（如 `:::writing`、`:::code_execution`、`:::search_results`）时，系统会自动将其渲染为带样式容器和复制按钮的格式化内容。这类结构常被较新的 OpenAI 模型用于区分正文、代码结果、搜索结果等不同类型输出。

- **结构化回复编辑**：点击包含工具调用、推理块或代码解释器输出的助手回复旁的编辑图标，将打开一个结构化编辑器，而非普通的纯文本输入框——每个输出项（文本、思考、工具调用、代码块、网络/文件/计算机搜索）都会成为独立的可折叠字段，同时 **JSON** 切换开关可暴露底层 [OpenAI Responses API](https://platform.openai.com/docs/api-reference/responses) 载荷以供直接编辑。这里的推理/思考块也可编辑——之前的纯文本编辑器会将其简化成不可编辑的 `<details>` 占位符。仅含纯文本的回复则保持原有的文本框行为。编辑会重写消息的 `output` 数组；保存时 Open WebUI 会重新推导显示的 `content`（包括思考和工具调用的 `<details>` 块），因此渲染的消息始终保持一致。
