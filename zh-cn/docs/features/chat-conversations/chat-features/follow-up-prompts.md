## 追问提示

Open WebUI 可以在每次模型响应后自动生成后续问题建议。这些建议会以可点击的标签形式显示在回复下方，帮助你无需重新输入提示词，就能进一步探索话题。

## 设置

你可以在 **设置 > 界面** 的 **聊天** 区域中配置追问提示行为：

### 自动生成追问

#### 默认：开启

会在每次模型响应后自动生成后续问题建议。这些建议由 [任务模型](/reference/env-configuration#task_model) 根据当前对话上下文生成。

- **开启**：每次模型响应后都会生成追问提示
- **关闭**：不生成任何追问建议

### 在聊天中保留追问提示

#### 默认：关闭

默认情况下，追问提示只会出现在最近一条消息下方；当你继续对话后，它们就会消失。

- **开启**：追问提示会被保留，并持续显示在聊天历史中的所有消息下
- **关闭**：只有最后一条消息会显示追问提示

:::tip 非常适合知识探索
当你在探索知识库时，可以开启此设置。这样你能看到之前每条回复对应的所有推荐追问，便于回溯并尝试不同的信息探索路径。
:::

### 将追问插入输入框

#### 默认：关闭

控制你点击追问提示后会发生什么。

- **开启**：点击追问后，会先把文本插入输入框，你可先编辑再发送
- **关闭**：点击追问后，会立即作为下一条消息发送

## 重新生成追问

如果你想为某条特定回复重新生成追问建议，可以使用社区中的 [重新生成追问](https://openwebui.com/posts/9b5ac6d6-dfd6-4cad-bc1d-5518b138f22d) action button。
