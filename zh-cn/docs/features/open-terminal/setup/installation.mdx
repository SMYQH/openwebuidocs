# 安装

根据你的需求选择安装方式：

| 方式 | 适用场景 |
| :--- | :--- |
| **Docker** | 在隔离容器中运行，与宿主系统完全分离。 |
| **裸机** | AI 直接在你的机器上工作——使用真实的文件、工具和环境。 |
| **Docker Compose** | 你已在使用 Docker Compose 运行 Open WebUI，并希望将 Open Terminal 添加进去。 |

在下方选择你的方式，然后按步骤操作。

<Tabs>
  <TabItem value="docker" label="Docker" default>

需要安装 [Docker](https://www.docker.com/products/docker-desktop/)。

### 1. 启动容器

```bash
docker run -d \
  --name open-terminal \
  --restart unless-stopped \
  -p 8000:8000 \
  -v open-terminal:/home/user \
  -e OPEN_TERMINAL_API_KEY=your-secret-key \
  ghcr.io/open-webui/open-terminal
```

将 `your-secret-key` 替换为你自己的密码。

<summary><b>参数说明</b></summary>

| 参数 | 用途 |
| :--- | :--- |
| `-d` | 在后台运行 |
| `--name open-terminal` | 容器名称，便于引用 |
| `--restart unless-stopped` | 重启时自动启动 |
| `-p 8000:8000` | 在 `http://localhost:8000` 暴露服务 |
| `-v open-terminal:/home/user` | 在重启间保持文件持久化 |
| `-e OPEN_TERMINAL_API_KEY=...` | 认证密钥 |

</details>

### 2. 验证

在浏览器中打开 `http://localhost:8000/docs`，你应该能看到 Open Terminal API 页面。

:::tip 自动生成的 API Key
如果未设置 API Key，系统会自动生成一个。通过以下命令查看：
```bash
docker logs open-terminal
```
:::

:::note OpenShift
对于 OpenShift 受限 SCC，请通过 Terminals 使用 `ghcr.io/open-webui/open-terminal:openshift`。该镜像支持受限的非 root Pod，但不支持运行时软件包安装、Docker 套接字访问、iptables 出站防火墙或 `OPEN_TERMINAL_MULTI_USER=true`。
:::

### 1. 运行 Open Terminal

**方式 A — 免安装直接运行：**

```bash
uvx open-terminal run --host 0.0.0.0 --port 8000 --api-key your-secret-key
```

通过 [uv](https://docs.astral.sh/uv/) 下载并运行，无需永久安装。

**方式 B — 通过 pip 安装：**

```bash
pip install open-terminal
open-terminal run --host 0.0.0.0 --port 8000 --api-key your-secret-key
```

{/* TODO: Screenshot — Terminal showing Open Terminal starting up with server address and API key. */}

### 2. 工作目录

Open Terminal 从当前目录提供文件服务。若要将其限定在特定项目：

```bash
cd /path/to/your/project
uvx open-terminal run --host 0.0.0.0 --port 8000 --api-key your-secret-key
```

或者跳过 `cd`，使用 `--cwd` 标志将服务器指向项目：

```bash
uvx open-terminal run --cwd /path/to/your/project --host 0.0.0.0 --port 8000 --api-key your-secret-key
open-terminal run --cwd /path/to/your/project
```

### Windows

通过 [pywinpty](https://github.com/andfoy/pywinpty) 支持 Windows。完整支持 PowerShell，包括彩色输出和交互式程序。

  <TabItem value="compose" label="Docker Compose">

在你现有的 `docker-compose.yml` 中添加 Open Terminal 服务：

### 1. 更新 Compose 文件

```yaml title="docker-compose.yml"
services:
  open-webui:
    image: ghcr.io/open-webui/open-webui:latest
    container_name: open-webui
    ports:
      - "3000:8080"
    volumes:
      - open-webui:/app/backend/data

  open-terminal:
    image: ghcr.io/open-webui/open-terminal
    container_name: open-terminal
    volumes:
      - open-terminal:/home/user
    environment:
      - OPEN_TERMINAL_API_KEY=your-secret-key

volumes:
  open-webui:
  open-terminal:
```

### 2. 启动服务

```bash
docker compose up -d
```

:::tip 连接 URL
当两个服务共享同一个 Docker Compose 网络时，使用服务名称作为主机：
```
http://open-terminal:8000
```
`localhost` 在容器之间无法解析。

## 作为 MCP 服务器运行

Open Terminal 也可以作为 [Model Context Protocol](https://modelcontextprotocol.io/) 服务器运行，使用 `open-terminal mcp` 将终端暴露给 MCP 兼容客户端。这需要可选的 `mcp` 额外依赖（会引入 [FastMCP](https://github.com/jlowin/fastmcp)）。

**方式 A — 免安装直接运行：**

```bash
uvx --from "open-terminal[mcp]" open-terminal mcp
```

**方式 B — 通过 pip 安装：**

```bash
pip install "open-terminal[mcp]"
open-terminal mcp
```

默认情况下，服务器使用 `stdio` 传输方式（客户端启动进程并通过标准输入/输出进行通信），因此大多数 MCP 客户端只需上述命令即可。

如需通过 HTTP 提供服务，可切换到 `streamable-http` 传输方式。`--host` 和 `--port` 标志仅适用于此传输方式（默认值：host `0.0.0.0`，port `8000`）：

```bash
open-terminal mcp --transport streamable-http --host 127.0.0.1 --port 8000
```

`--config`（TOML 配置文件路径）和 `--cwd`（服务器进程的工作目录）标志与 `open-terminal run` 的工作方式相同。
