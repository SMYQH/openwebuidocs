# 使用 Open WebUI 能做什么

**面向所有 AI 模型的统一界面。私有、可扩展，专为团队构建。**

Open WebUI 旨在取代团队日常繁杂的各种 AI 工具——用来写作的 ChatGPT、独立运行的图像生成应用、专用的文档搜索工具、以及塞满提示词却无人问津的电子表格。现在，所有内容都汇聚于此：对话、知识库、工具，以及驱动它们的各个模型。

---

## 💬 聊天与对话

**与任何模型对话，中途无缝切换，并保持完整的上下文。**

对话是 Open WebUI 的核心。在一个统一界面中与 Ollama、OpenAI、Anthropic 或任何兼容 OpenAI 接口的提供商进行聊天。附带文件、搜索网页、执行代码、让 AI 调用工具——这一切都无需离开聊天界面。

| | |
| :--- | :--- |
| 🔀 **多模型聊天** | 并排运行两个模型并比较它们的响应 |
| 📎 **文件与图像上传** | 附加文档、图像和代码供 AI 分析 |
| 🔍 **网页搜索** | AI 实时搜索网页并引用信息来源 |
| 🐍 **代码执行** | 直接在浏览器中运行 Python，或通过 [终端](/features/open-terminal) 运行 |
| 📝 **消息队列** | 在 AI 回复时继续输入——消息将自动排队发送 |
| 🧠 **记忆** | AI 能够跨对话记住关于你的事实 |
| 🗂️ **文件夹、标签、固定** | 按照你的工作方式组织对话 |
| 🎤 **语音与音频** | 语音转文本、文本转语音、免提语音与视频通话 |
| 🖼️ **图像生成** | 使用 DALL-E、Gemini、ComfyUI 等生成和编辑图像 |
| ⏱️ **自动化** | 安排提示词在设定的循环时间自动运行 |
| ✅ **任务管理** | 模型可维护结构化任务列表以处理多步骤工作流 |

## 📚 知识库与 RAG

**让你的 AI 访问文档——并让它自己找到重点。**

上传文件、构建知识库，让 AI 准确检索所需信息。在面对海量集合时选择向量搜索（RAG），或者在需要精确度时注入完整内容。借助原生的函数调用功能，模型能够自主在你的整个知识库中搜索、浏览并整合信息。

| | |
| :--- | :--- |
| 📄 **9 种向量数据库** | 官方维护 ChromaDB 和 PGVector，另有社区支持选项：Qdrant、Milvus、Elasticsearch 等 |
| 🔍 **混合搜索** | BM25 + 带有交叉编码器重排的向量搜索 |
| 📑 **5 种提取引擎** | Tika、Docling、Azure、Mistral OCR 以及自定义加载器 |
| 🤖 **智能体检索** | 模型自主搜索并阅读你的文档 |
| 📄 **完整上下文模式** | 注入完整文档——无需切块，无需猜测 |
| 🔄 **知识库同步** | 通过 [oikb](/ecosystem/knowledge-base-sync) 让 KB 与文件夹、Git 仓库、Wiki 或存储桶保持同步 |

[**了解知识库 →**](/features/workspace/knowledge) · [**知识库同步 →**](/ecosystem/knowledge-base-sync)

## 🤖 模型与智能体

**用自定义指令、工具和知识库包装任何模型，打造专属智能体。**

创建一个始终遵循你的代码规范的“Python 导师”。一个带有公司模板的“会议总结助手”。一个内置代码检查规则的“代码审查员”。每个智能体都是一个配置包装器——只需选择任意基础模型，绑定知识库、工具和系统提示词即可。

| | |
| :--- | :--- |
| 🧩 **模型预设** | 将系统提示词、工具、知识库和参数打包为一体 |
| 🏷️ **动态变量** | 自动注入 `{{ USER_NAME }}`、`{{ CURRENT_DATE }}` 等变量 |
| 🔧 **绑定工具** | 为特定模型强制启用特定工具 |
| 👥 **访问控制** | 限制特定用户或群组对模型的访问权限 |
| 📊 **全局默认值** | 为所有模型设置基础能力和参数 |

[**了解模型 →**](/features/workspace/models)

## 📝 笔记

**在 AI 的陪伴下写作、思考和完善。**

这是一个专门为不属于单一对话的内容提供的专用工作区。使用富文本编辑器起草，让 AI 原地重写文本，或将笔记作为精确的上下文注入任何聊天中——无需分块，无需向量检索。

| | |
| :--- | :--- |
| ✍️ **富文本编辑器** | 支持 Markdown 和富文本，带有悬浮格式化工具栏 |
| 🤖 **AI 增强** | 原地重写或改进所选文本 |
| 📎 **上下文注入** | 将笔记附加到任何聊天中，提供全保真的上下文 |
| 🔍 **智能体访问** | 模型可自主搜索、阅读并更新笔记 |

[**了解笔记 →**](/features/notes)

## 💬 频道

**你的团队和 AI 在此实时共同思考。**

这是一个持久的共享空间，人类和 AI 模型可以在同一个对话中互动。呼叫 `@gpt-4o` 起草计划，然后呼叫 `@claude` 进行点评——你的整个团队可以在同一时间线上看到这两者的回复。

| | |
| :--- | :--- |
| 🤖 **@模型 标签** | 随时按需将任何 AI 模型召唤到对话中 |
| 🧵 **话题与互动** | 回复、固定和表情符号互动 |
| 🔒 **访问控制** | 公共、私有、基于群组和直接消息频道 |
| 🧠 **AI 频道感知** | 模型可自主跨频道搜索并综合信息 |

[**了解频道 →**](/features/channels)

## ⚡ 终端

**为你的 AI 提供一台真正的计算机去工作。**

将一个真正的计算环境连接到 Open WebUI。AI 可以编写代码、执行代码、读取输出、修复错误并进行迭代——这一切都在聊天中完成。它可以处理文件、安装包、运行服务器并返回结果。

| | |
| :--- | :--- |
| 🖥️ **代码执行** | 运行真实的命令并返回输出 |
| 📁 **文件浏览器** | 在侧边栏中浏览、上传、下载和编辑文件 |
| 🌐 **网站预览** | 在 Open WebUI 中实时预览 Web 项目 |
| 🔒 **可选的隔离机制** | Docker 容器或裸机 |

[**了解终端 →**](/features/open-terminal)

:::tip
想在浏览器标签页里拥有整台机器？请看 [**cptr**](https://github.com/open-webui/computer)（Open WebUI Computer）。
:::

## 🔌 扩展能力

**通过 Python 工具、管道、MCP 或 OpenAPI 服务器添加任何功能。**

Open WebUI 是一个平台，而不是一个封闭的产品。编写可以在聊天中运行的 Python 工具。通过 OpenAPI 或 MCP 连接外部服务。构建可以过滤、转换或路由每条消息的 Pipeline。一键导入社区构建的扩展程序。

| | |
| :--- | :--- |
| 🐍 **工具与函数** | 在聊天中运行的 Python 脚本——包含内置代码编辑器 |
| 🔧 **管道** | 用于过滤器、提供商和自定义逻辑的模块化插件框架 |
| 🔗 **MCP 支持** | 原生支持模型上下文协议 (Model Context Protocol) 服务器的流式 HTTP |
| 🌐 **OpenAPI 服务器** | 自动发现并接入任何兼容 OpenAPI 端点的工具 |
| 📝 **技能** | Markdown 指令集，教导模型如何处理特定任务 |
| ⚡ **提示词** | 带有类型化输入变量和版本控制的斜杠命令模板 |

[**了解扩展能力 →**](/features/extensibility)

## 🔐 认证与访问

**控制谁能进入、他们能做什么，以及如何连接到你的身份系统。**

Open WebUI 从第一天起就支持多用户。定义角色、创建用户组、设置每个模型的访问权限，并与你的身份提供商集成。从单人安装到数千席位的组织均可适用。

| | |
| :--- | :--- |
| 👥 **RBAC** | 角色、组和针对特定资源的权限控制 |
| 🔐 **SSO/OIDC/LDAP** | 与任何身份提供商联合认证 |
| 📋 **SCIM 2.0** | 自动化的用户和群组预配 |
| 🔑 **API 密钥** | 脚本、机器人和集成的编程式访问 |

[**了解认证与访问 →**](/features/authentication-access)
