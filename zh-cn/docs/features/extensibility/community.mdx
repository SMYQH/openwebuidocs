[Open WebUI 社区网站](https://openwebui.com/) 上有数百个社区构建的工具和函数，你可以一键导入到你的实例中。本页面是初学者的地图：库的工作原理，如何根据你想要的功能选择插件，以及如何安全导入。

:::danger 高亮不代表经过审查

在本页面或 Open WebUI 社区网站上被标记为**精选、热门或列出，并不代表认可**，也**不**意味着代码已经过安全或质量审查。列表是社区提交的，未经审计。

工具和函数在你的服务器上执行**任意 Python 代码**。插件的星数或排名无法说明其代码的实际行为。每次导入前请阅读源代码。请参阅完整的[插件安全警告](/features/extensibility/plugin/)。

:::

## 库的工作原理

- **浏览** openwebui.com 上的[工具和函数](https://openwebui.com/)，或按关键字[搜索](https://openwebui.com/search)。
- **一键导入**（或通过将源代码粘贴到内置编辑器中）。无需 `pip install`，无需重启。
- **导入后的位置：** 导入的**工具**出现在**工作区 → 工具**下，导入的**函数**出现在**管理面板 → 函数**下。两者均由管理员管理。关于两者的区别，请参见[工具与函数](/features/extensibility/plugin/)。

## 根据你想要的功能查找插件

插件*类型*并不总是从名称中明显看出，所以从目标开始。在[社区网站](https://openwebui.com/search)上搜索以下术语，然后在导入前阅读代码。

| 我想要…… | 插件类型 | 在社区网站搜索 |
| :--- | :--- | :--- |
| 给模型**实时网络搜索**或实时数据 | [工具](/features/extensibility/plugin/tools) | `web search`、`weather`、`search` |
| **添加模型提供商**（Anthropic、Gemini、路由器、本地引擎） | [Pipe](/features/extensibility/plugin/functions/pipe) | `pipe`、提供商名称 |
| **跟踪或限制每个用户的成本/使用量** | [Filter](/features/extensibility/plugin/functions/filter) | `cost`、`usage`、`budget`、`rate limit` |
| **编辑 PII** 或审核消息内容 | [Filter](/features/extensibility/plugin/functions/filter) | `pii`、`redact`、`moderation` |
| **总结或压缩**长对话 | [Filter](/features/extensibility/plugin/functions/filter) | `context`、`memory`、`compaction` |
| 在消息上添加**按钮**（导出、重新运行、翻译） | [Action](/features/extensibility/plugin/functions/action) | `action`、任务名称 |
| **响应系统事件**（审计日志、注册门控、资源预配） | [Event](/features/extensibility/plugin/functions/event) | `event`、`webhook`、`audit` |
| 连接**现有的 HTTP 服务** | [OpenAPI / MCP 服务器](/features/extensibility/mcp) | 不适用（将 Open WebUI 指向规范即可） |

不确定哪种类型适合？[我需要哪种扩展？](/features/extensibility#which-extension-do-i-need) 表格会帮你理清。

## 导入前的初学者安全检查清单

社区插件与服务器其他部分具有相同的访问权限。安装社区插件就像以管理员身份运行他人的脚本。在导入之前：

1. **阅读代码。** 如果你不理解它的功能，就不要安装。特别警惕对未知主机的网络调用、读取你的 `data` 目录或环境变量的代码，以及任何混淆的代码。
2. **检查它是否只做声称的事情。** 一个"翻译"操作没有理由回传数据或读取你的数据库。
3. **查看 `requirements:`。** 额外的 pip 依赖在加载时安装。未知的包是供应链风险，在多副本部署中，运行时安装会在工作节点间竞争（设置 [`ENABLE_PIP_INSTALL_FRONTMATTER_REQUIREMENTS=False`](/reference/env-configuration#enable_pip_install_frontmatter_requirements) 并将其烘焙到你的镜像中）。
4. **先在非生产实例上尝试**，确认可信后再推广。
5. **限制谁能导入。** 只有受信任的管理员才能创建或导入插件。除非必要，普通用户不应拥有工作区访问权限。

关于插件加载后能访问什么（以及随之而来的风险），请参见[底层机制](/features/extensibility/plugin/development/under-the-hood)。

## 分享你自己的插件

构建了有用的东西？将其发布到社区网站，以便其他人可以找到并导入它。同样的警告也适用于你：清晰的 README 和可读、可审计的代码才能赢得信任，因为没有人会替你审计。将密钥放在[阀门](/features/extensibility/plugin/development/valves)中而非硬编码，并编写 `async` 处理程序，以便你的插件在后端向完全异步执行演进时保持兼容。
