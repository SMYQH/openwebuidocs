# Open WebUI

**Open WebUI 是一个[可扩展](/features/extensibility/plugin/)、功能丰富且易于使用的自托管 AI 平台，专为完全离线运行而设计。** 它支持 **Ollama** 和 **兼容 OpenAI 的 API**，无论是本地模型还是云端模型，都是一套强大且与提供商无关的解决方案。

## 快速开始

<Tabs>
  <TabItem value="docker" label="Docker" default>

```bash
docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main
```

然后打开 [http://localhost:3000](http://localhost:3000)。

如需 GPU 支持、Docker Compose 等更多内容 → [完整 Docker 指南](/getting-started/quick-start)

  </TabItem>
  <TabItem value="pip" label="pip">

```bash
pip install open-webui
open-webui serve
```

然后打开 [http://localhost:8080](http://localhost:8080)。

  </TabItem>
  <TabItem value="uv" label="uv">

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
DATA_DIR=~/.open-webui uvx --python 3.11 open-webui@latest serve
```

然后打开 [http://localhost:8080](http://localhost:8080)。

  </TabItem>
  <TabItem value="desktop" label="桌面端">

从 [**github.com/open-webui/desktop**](https://github.com/open-webui/desktop) 下载桌面应用。它可在你的系统上原生运行 Open WebUI，无需 Docker 或手动配置。

:::info
用于生产环境时，请通过 **Docker** 或 **Python** 安装。
:::

  </TabItem>
</Tabs>

## Open WebUI Computer

[**Open WebUI Computer**](/ecosystem/computer) 是一个在你的真实机器上运行的代理框架。安装一个应用，加入 AI（API 密钥、本地模型，或你已有的 Claude Code、Codex 等编码代理订阅），它就能处理你真实的文件和项目——从研究、写作到编码，附带审批和计划模式。它不会启动一次性的沙箱然后丢给你一个 diff。代理编辑的文件就是你磁盘上的文件。

因为它是你的整台机器，其他功能也随之免费获得：从任何浏览器（桌面或手机）访问文件、编辑器、终端和 git；一个你可以通过 Telegram 或 WhatsApp 联系它的助手；定时代理向你报告。一切都在你拥有的硬件上本地运行，无需托管在别处。

Computer 和 Open WebUI 是一个生态系统，按设计以两种速度运行。**Open WebUI** 是稳定平台：企业级、多用户，是团队和组织部署和信赖的平台。**Computer** 迭代更快：新能力优先在此构建和发布，经过日常使用打磨后，再融入 Open WebUI 中。如果你想要我们正在构建的最前沿功能，它会先登陆 Computer。

- 刚接触？[什么是 Computer？](/ecosystem/computer/what-is-computer) · [快速开始](/ecosystem/computer/quickstart) · [使用场景](/ecosystem/computer/use-cases/)
- 已经在用 Open WebUI？[将 Computer 工作空间作为模型接入 Open WebUI](/ecosystem/computer/automate/open-webui)

## 其他 Open WebUI 项目

- [**open-terminal**](https://github.com/open-webui/open-terminal) — 给你的 AI 一台真正的电脑。为 Open WebUI 提供沙箱化的终端、文件浏览器和代码执行能力。
- [**oikb**](https://github.com/open-webui/oikb) — 让你的 Open WebUI 知识库保持同步。可监控本地文件夹、GitHub 仓库、S3 存储桶、Confluence 以及其他 40+ 种数据源。
- [**mcpo**](https://github.com/open-webui/mcpo) — MCP 转 OpenAPI 代理。在 Open WebUI（或任何 OpenAPI 客户端）中使用任意 MCP 工具服务器，无需自写胶水代码。

## 面向 LLM 与编码代理

本文档的机器可读入口点：

- [llms.txt](/llms.txt)：每篇文档页面的精选索引，附简短描述。
- [llms-full.txt](/llms-full.txt)：所有文档页面合并为单个 Markdown 文件。
- [agents.txt](/agents.txt)：供使用本文档的代理参考的简洁说明。
- [api/search?q=YOUR_QUERY](https://docs.openwebui.com/api/search?q=YOUR_QUERY)：确定性文档搜索。

所有文件在每次部署时都会重新生成。
