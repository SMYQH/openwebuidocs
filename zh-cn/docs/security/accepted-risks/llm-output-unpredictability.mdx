# LLM 输出不可预测性

|  |  |
| :--- | :--- |
| **类别** | 模型固有问题（提示注入、越狱、不安全或幻觉输出）— OWASP LLM Top 10 |
| **状态** | LLM 固有问题；不是可修补的应用缺陷；单独发生时超出漏洞范围，除非作为与合法 Open WebUI 漏洞链式使用的一部分经证实（参见[安全策略](../security-policy)） |

---

## 概述

Open WebUI 尊重并赞赏 [OWASP LLM 应用 Top 10](https://owasp.org/www-project-top-10-for-large-language-model-applications/)；它是思考 AI 系统如何失败的有价值框架。然而，它编录的许多行为并非经典安全漏洞。它们是大型语言模型自身的固有风险，由任何构建在 LLM 之上的应用程序承载，而非由特定供应商代码中的缺陷引入。当这些模型被赋予工具和自主权时，相同的风险延伸至智能体行为；请参见[智能体应用风险](./agentic-application-risks)。

大型语言模型是非确定性的，且可被其输入操纵。模型可能通过精心设计的提示、获取的网页或上传文档中的注入内容，或对抗性对话，被说服产生不安全、误导性或攻击者期望的输出：越狱、提示注入、幻觉事实、不安全建议，或尝试以用户未预期的方式使用可用工具。

这种不可预测性是语言模型工作方式的属性。它**不是 Open WebUI 代码的缺陷**。

## 我们缓解的措施

Open WebUI 控制模型**周围**的系统，我们加固该表面：

- **访问控制和隔离**限制了模型及其工具可触及的范围：按用户的数据范围、[工具和函数权限模型](/features/extensibility/plugin/)以及容器隔离的代码执行。
- **模型不能超越调用用户自身的权限。** 代表用户行动时，它受该用户权限约束；它不能读取或执行用户无法做到的事情。
- **操作员护栏**（[过滤器](/features/extensibility/plugin/functions/filter)、内容检查、确认提示）让部署可以添加自己的策略层。

这些措施减少了不良模型输出的**后果**。它们不能也无法使模型的输出本身在每种情况下都可预测或安全。

## 剩余部分及为何不被视为漏洞

其全部影响为**模型的输出**的安全问题——模型被说服说了什么、推荐了什么或尝试了它已被允许尝试的行动——是**LLM 固有的，且单独而言不是 Open WebUI 漏洞**。它受限于模型对齐的技术现状，而非 Open WebUI 代码可修补的内容，且"使模型永不产生有害输出"不是可实现的应用级修复。

因此，**当全部影响仅为模型产生不期望的内容，或被说服误用其已被允许使用的能力时，我们不接受作为漏洞的报告。** 该行为是运行 LLM 的已接受固有属性，不会作为缺陷处理。

**这是关于模型行为的范围声明，而非全面豁免。** 一旦精心设计的输入从"模型说了什么"跨越到 Open WebUI 自身代码或控制中的具体缺陷，报告**就属于范围**内，并被视作真实漏洞处理，例如：

- 提示注入**通过 Open WebUI 代码路径升级为代码执行、数据访问或权限提升**；
- **绕过访问控制、权限或隔离检查**；或
- 模型仅为底层应用错误的**触发**因素的任何缺陷。

这些是 Open WebUI 的缺陷，并会作为缺陷进行修复。完整报告范围请参见[安全策略](../security-policy)。
