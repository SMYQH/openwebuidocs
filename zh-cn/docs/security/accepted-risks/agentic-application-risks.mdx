# 智能体应用风险

|  |  |
| :--- | :--- |
| **类别** | 智能体系统固有问题（目标劫持、工具误用、记忆与上下文投毒、欺骗性或恶意多智能体行为），映射至 OWASP 智能体应用 Top 10 |
| **状态** | 自主 AI 智能体固有问题；单独发生时不是可修补的应用缺陷；除非作为与合法 Open WebUI 漏洞链式使用的一部分经证实，否则超出漏洞范围（参见[安全策略](../security-policy)） |

---

## 概述

Open WebUI 尊重并赞赏 [OWASP 智能体应用 Top 10](https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/)；它是思考自主、使用工具的 AI 系统如何失败的有价值框架。然而，它编录的许多行为并非经典安全漏洞。它们是赋予语言模型自主权所带来的固有风险，由任何允许模型规划、使用工具和行动的应用程序承载，而非由特定供应商代码中的缺陷引入。这些风险直接建立在 [LLM 输出不可预测性](./llm-output-unpredictability) 中涵盖的模型级行为之上，两者高度重叠：智能体是赋予工具和自主权的语言模型，因此相同的可操纵性既可产生不安全的输出，也可驱动不安全的行动。

智能体系统进行规划、决策、调用工具并在步骤间携带上下文，且它们基于不受信任的自然语言输入进行这些操作。因此，智能体可能被精心设计的提示、投毒的文档或网页、欺骗性的工具输出或来自对等智能体的消息所操纵，从而追求被劫持的目标、误用已授予的工具、基于中毒的记忆行事，或说服人类批准不应批准的事项。这些是自主 LLM 智能体工作方式的属性。它们**不是 Open WebUI 代码的缺陷**。

## 我们缓解的措施

Open WebUI 的角色是提供平台并控制智能体被**允许**执行的操作。我们有意义的控制是用户门禁权限、特权限制、访问控制和隔离：

- **功能受到门禁，默认不授予。** 智能体只能通过工具和函数行动。工具使用受 `workspace.tools` 权限控制，该权限**为非管理员用户默认禁用**，函数创作仅限于管理员。授予其中任何一个在文档中均等同于向该用户授予服务器的 shell 访问权限（参见[安全策略](../security-policy)规则 10）。
- **智能体不能超越调用用户自身的权限。** 代表用户行动时，它受该用户的角色和 RBAC 权限以及按用户的数据范围约束；它不能读取或访问用户无法触及的内容。
- **执行隔离。** 服务器端代码执行是可选且由管理员配置的，对于生产环境，[Terminals 编排器](/features/open-terminal/terminals)在每个会话、容器隔离的环境中运行执行，并配置了可自定义的[安全上下文和资源策略](/features/open-terminal/terminals/orchestration/policies)。
- **操作员护栏。** [过滤器](/features/extensibility/plugin/functions/filter)、人工确认和内容检查让部署可以围绕智能体可执行的操作添加自己的策略层。

这些控制约束了被操纵或行为不当的智能体的**后果**。它们不能也无法使自主智能体的决策在每种情况下都可靠安全。

## 剩余部分及为何不被视为漏洞

其全部影响为**智能体以智能体固有可能的方式行为**（被劫持的目标、中毒的记忆、在已授予权限内不安全的工具使用、被自信的智能体说服的人类）的风险**是自主智能体固有的，且单独而言不是 Open WebUI 漏洞**。它受限于模型和智能体对齐的技术现状，而非 Open WebUI 代码可修补的内容。

因此，**当全部影响仅为智能体被说服误用其已被允许使用的能力，或在其被授予的范围内产生不期望的输出或行动时，我们不接受作为漏洞的报告。** 这是运行自主智能体的已接受固有属性。

**这是关于智能体行为的范围声明，而非全面豁免。** 一旦精心设计的输入从"智能体被操纵"跨越到 Open WebUI 自身代码或控制中的具体缺陷，报告**就属于范围**内，并被视作真实漏洞处理，例如：

- 智能体行动**通过 Open WebUI 代码路径提升权限、访问另一用户的数据或获得代码执行能力**，而非通过操作员授予的能力；
- **绕过访问控制、权限或隔离检查**（例如工具访问调用者无权访问的数据）；或
- 智能体仅为底层应用错误的**触发**因素的任何缺陷。

这些是 Open WebUI 的缺陷，并会作为缺陷进行修复。此区分镜像了框架自身的区分：智能体严格在其授予权限内误用工具是固有的智能体风险，而通过平台缺陷的权限提升或代码执行是缺陷。完整报告范围请参见[安全策略](../security-policy)。
