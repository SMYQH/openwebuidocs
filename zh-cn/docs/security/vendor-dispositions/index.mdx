# 供应商处置

**Open WebUI 对外部报告的 CVE 和安全声明的正式评估。**

当针对 Open WebUI 提交 CVE 时，我们会根据我们的 [安全策略](../security-policy)、我们记录的 [威胁模型](#威胁模型) 和软件的实际行为来评估报告。如果报告不准确、误导性或不代表真正的漏洞，我们会在此发布供应商处置以解释我们的评估。

:::note 自动化扫描器标志
关于自动化供应链扫描器指标（socket.dev 及类似工具）而非特定 CVE 的立场，请参见[供应链与安全扫描器](../supply-chain-security)。
:::

---

## 什么是供应商处置？

**供应商处置**是软件供应商对 CVE 或漏洞报告的官方公开回应。它是协调漏洞披露流程的标准部分，具有多个目的：

- **透明度**：用户和管理员可以确切看到我们如何评估一项声明以及为何得出结论。
- **准确性**：CVE 数据库有时包含错误描述严重程度、误解威胁模型或将预期行为描述为漏洞的报告。处置可纠正公开记录。
- **指导**：每份处置都包含背景信息，帮助管理员评估报告的问题是否与其特定部署相关。

---

## 威胁模型

Open WebUI 是一个**自托管、单租户、经过身份验证、基于角色**的应用程序。它可以公开部署，但需要适当配置（TLS 终止、反向代理、适当的访问控制）。我们的威胁模型假设：

- 所有用户在访问任何功能前都经过**身份验证**。
- **管理员**是受信任的行为者，拥有完整的系统控制权。
- **安全设置处于默认值或更严格的配置下**：威胁模型不假设故意削弱安全设置。有效的漏洞报告必须在默认或更严格的配置下可重现。
- **工具、函数和管道**按设计执行任意代码：这是一个功能，而非漏洞。管理员控制谁可以安装和使用它们。

忽略这些架构假设的报告——例如，声称需要管理员级别访问、故意削弱设置或配置错误部署的漏洞——可能会被争议。

如需完整概述，请参阅[安全策略](../security-policy)。
