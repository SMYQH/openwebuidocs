# Public Storage 使用 Open WebUI 在所有企业职能部门扩展安全的冠军驱动 AI 平台

![Public Storage](/images/enterprise/customers/public-storage/hero.png)

## 概述

:::info
Public Storage 如何通过**冠军驱动的推出方式**在 GCP 上部署安全的私有生成式 AI 平台（使用 **Open WebUI**），在 **30 天内达到约 50% 的活跃采用率**，并将关键操作工作流的时间减少**超过 80%**。

### 一览

- **用户数**：5,000-10,000 名员工
- **地区**：美国（强制执行数据驻留）
- **行业**：房地产
- **部署**：GCP（容器化、私有网络）
- **模型**：Anthropic Claude（主要）、OpenAI GPT（高流量）、Llama、Gemma（开源权重）
- **部署时间**：2 周试点，30 天内完全可用
- **采用率**：第一个月约 50% 活跃使用率
- **关键成果**：操作工作流显著节省时间、跨职能部门 AI 赋能、企业级治理

## 关于 Public Storage

[Public Storage](https://www.publicstorage.com/) 是一家财富 500 强自存储房地产投资信托基金，在美国运营 3,000 多处设施。拥有数百万客户和数千名员工，跨越公司办公室、呼叫中心和现场运营，该公司管理着世界上最大的房地产组合之一。这种运营足迹使得对 AI 的一致、受治理的访问成为有意义的竞争优势。

## 挑战：在企业范围内安全地扩展 AI

随着生成式 AI 工具的普及，Public Storage 认识到了机遇和风险。整个组织的团队开始独立试验 AI，创建了零散的临时订阅和无治理使用的局面。

领导层设定了明确的目标：提供**安全的集中式 AI 平台**，提高整个组织的 AI 素养，显示高价值的商业使用案例，并实现可重复的生产力收益，同时不影响数据安全或合规性。

**关键需求**

- **私有部署**：在企业 GCP 环境内
- **访问限制**：仅限于企业防火墙后的已认证员工
- **SSO 集成**：与现有身份提供商 **Okta** 集成
- **模型灵活性**：无供应商或单一 LLM 锁定
- **完整审计跟踪**、DLP 政策和 PII 脱敏
- **基于角色的访问控制**：按群组划分

基于 SaaS 的 AI 工具提供了便利性，但缺乏 Public Storage 所需的隐私控制、网络限制和模型灵活性。

## 解决方案：GCP 上的 Open WebUI

选择 Open WebUI 是因为其**可扩展的基础**：开放的架构允许 Public Storage 从基本的聊天界面演变为安全的多模型内部 AI 平台。它支持快速迭代、广泛的模型选择和企业治理，同时保持完全私有。

**架构亮点**

- **计算/编排**：GCP 上的容器化工作负载，根据内部需求自动扩展
- **存储/数据库**：托管 PostgreSQL 和安全对象存储（GCP 原生）
- **网络**：私有网络；访问限制在企业防火墙后的已认证员工；开源权重 LLM API 的私有端点
- **CI/CD**：用于受控 Open WebUI 升级的自动化管道
- **日志/监控**：通过 GCP 原生工具和 **Langfuse** 的集中日志和使用仪表板用于 LLM 可观测性
- **安全控制**：通过 IdP 的 MFA、基于群组的 RBAC、强制执行数据驻留、通过面向用户的中断进行 PII 脱敏、调节护栏、向 SIEM 的审计导出、DLP 政策、出站限制

> "我们的目标不仅是部署 AI，而是负责任地扩展它。Open WebUI 允许我们从业务中众包高价值的使用案例，同时保持所需的治理。" — Public Storage 首席技术官

## 模型和数据处理

Public Storage 采用**多模型策略**以平衡能力、隐私和成本：

- **主要**：Anthropic Claude（零数据保留）
- **高流量**：OpenAI GPT 模型（零数据保留）
- **开源权重**：Llama 和 Gemma 用于私有、基础设施工作负载
- **RAG**：默认 Sentence Transformers 用于嵌入，持续优化检索性能
- **PII 处理**：自定义过滤器**在任何敏感数据到达模型之前剥除 PII 并以面向用户的通知停止处理**。这种主动中断方法超越了静默脱敏

## 采用和赋能

Public Storage 采取了深思熟虑的、冠军驱动的推出方法。**为期两周的试点，配备指定的采用冠军**，在 30 天内进行了更广泛的启动，达到完全可用。

赋能专注于**实际、特定角色的使用案例**，而不是抽象的 AI 培训。团队为早期采用者进行了 1.5 小时的"PS.AI 定向"会议，分享了针对每个职能定制的真实示例和最佳实践。

在第一个月内：

- **约 50% 的已载入用户**在积极使用该平台
- 随着团队**与同行分享成功的工作流**，使用量继续增长
- **所有企业职能部门**都得到了代表，包括 HR、市场营销、财务、法律、呼叫中心、运营、销售/收购、IT 和风险管理

> "我们看到来自业务而非仅 IT 构建的使用案例的真实操作时间节省，这加快了采用并交付了实际成果。" — Public Storage 数字技术副总裁

## 成果：生产力、采用和治理

### 01. 操作时间节省

整个组织的团队报告了显著的生产力提升。一个例子：区域经理**将四小时的周预订审查流程减少到约 40 分钟**，减少了超过 80%。

**第一个月约 50% 的活跃使用率**，采用由员工构建自定义模型和角色、跨部门共享业务案例机会而自发推动。

- **完全私有部署**，数据驻留在 GCP 内强制执行
- 内置 PII 脱敏、调节护栏、DLP 政策和审计导出确保**无摩擦合规**
- RBAC、MFA 和出站限制在整个平台提供**分层安全**

Open WebUI 因为能够随着 Public Storage 的野心而成长而脱颖而出：

- **可扩展的基础**：从聊天演变为完整的内部 AI 平台

- **企业级治理**：PII 脱敏、审计跟踪、RBAC、DLP
- **快速部署**：IT 开销最少
- **开放架构**：支持迭代，无供应商依赖

该平台使每个企业职能部门能够安全、大规模地发现和共享 AI 驱动的生产力提升。
