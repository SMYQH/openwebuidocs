## 概览

Open WebUI 可以轻松连接 **OpenAI** 和 **Azure OpenAI**。本指南会带你完成 API key 配置、正确端点填写与模型选择，让你马上开始聊天。

对于其他提供兼容 OpenAI API 的提供商（如 Google Gemini、Mistral、Groq、DeepSeek 等），请参见 **[兼容 OpenAI 的提供商](/getting-started/quick-start/connect-a-provider/starting-with-openai-compatible)** 指南。至于 Anthropic 的 Claude 模型，请参阅专门的 **[Anthropic（Claude）](/getting-started/quick-start/connect-a-provider/starting-with-anthropic)** 指南。

---

## 重要：我们支持的是协议，而不是厂商私有实现

Open WebUI 是一个**以协议为中心**的平台。虽然我们为 OpenAI 模型提供了一流支持，但本质上主要还是通过 **OpenAI 聊天补全 API 协议**来实现。

我们优先支持被几十家提供商共同采用的通用标准，并对 **[Open Responses](https://www.openresponses.org/)** 这类新兴标准提供实验性支持。关于这一设计思路的详细解释，请参见 **[协议支持 FAQ](/faq#q-why-doesnt-open-webui-natively-support-provider-xs-proprietary-api)**。

## 第 1 步：获取你的 OpenAI API key

- **OpenAI：** 在 [platform.openai.com/account/api-keys](https://platform.openai.com/account/api-keys) 获取
- **Azure OpenAI：** 在 [Azure Portal](https://portal.azure.com/) 中获取

---

## 第 2 步：在 Open WebUI 中添加 API 连接

当 Open WebUI 已经运行后：

1. 进入 ⚙️ **Admin Settings**。
2. 打开 **Connections > OpenAI > Manage**（查找扳手图标）。
3. 点击 ➕ **Add New Connection**。

<Tabs>
  <TabItem value="standard" label="OpenAI" default>

  *   **Connection Type**：External
  *   **URL**：`https://api.openai.com/v1`
  *   **API Key**：你的密钥（通常以 `sk-...` 开头）

  </TabItem>
  <TabItem value="azure" label="Azure OpenAI">

  适用于 Microsoft Azure OpenAI 部署。

  1. 找到 **Provider Type**，点击当前标记为 **OpenAI** 的按钮，将其切换为 **Azure OpenAI**。
  2. **URL**：你的 Azure Endpoint（例如 `https://my-resource.openai.azure.com`）。
  3. **API Version**：例如 `2024-02-15-preview`。
  4. **API Key**：你的 Azure API Key。
  5. **Model IDs (Deployments)**：你**必须**在此填写具体的 Deployment Name（例如 `my-gpt4-deployment`）。

  </TabItem>
</Tabs>

### 高级配置

*   **Model IDs (Filter)**：
    *   *默认（留空）*：自动检测提供商中的全部可用模型。
  *   *填写后*：作为**白名单** 生效。只有这里列出的模型 ID 会对用户可见，适合隐藏旧模型或高成本模型。

*   **Prefix ID**：
    *   如果你连接了多个提供商，而它们刚好存在同名模型（例如都提供 `llama3`），可在这里加上前缀（如 `groq/`）用于区分。模型将显示为 `groq/llama3`。

4. 点击 **Save** ✅。

这样你的凭据就会被安全保存。

:::tip 连接超时配置

如果你的 API 提供商响应较慢，或你遇到了超时问题，可以调整模型列表获取超时：

```bash
# 针对慢网络提高超时（默认 10 秒）
AIOHTTP_CLIENT_TIMEOUT_MODEL_LIST=15
```

如果你保存了一个不可达 URL，导致界面卡住，请参见 [模型列表加载问题](/troubleshooting/connection-error#️-model-list-loading-issues-slow-ui--unreachable-endpoints) 故障排查页面。

:::

## 第 3 步：开始使用模型

保存连接后，你就可以直接在 Open WebUI 内使用模型。

🧠 无需下载任何模型——只要在模型选择器中选择一个模型并开始聊天即可。只要提供商支持该模型，你就能通过其 API 立即使用它。

模型选择界面如下：

![OpenAI 模型选择器](/images/getting-started/quick-start/selector-openai.png)

只需选择 GPT-4、o3-mini 或提供商支持的任意兼容模型即可。
