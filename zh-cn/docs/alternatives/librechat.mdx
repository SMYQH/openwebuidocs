# Open WebUI & LibreChat

*最后更新时间：2026 年 5 月*

[LibreChat](https://www.librechat.ai/) 是我们在这个领域真正尊敬的项目之一。它提供多提供商聊天体验，具有强大的身份验证支持、并排模型比较和专注的功能集，能很好地处理基础知识。该项目采用 MIT 许可证，积极维护，Danny 和他身后的社区创建了一些坚实的成果。

[GitHub](https://github.com/danny-avila/LibreChat) · [MIT License](https://github.com/danny-avila/LibreChat/blob/main/LICENSE)

---

## LibreChat 的优势

- **多提供商聊天**，为 OpenAI、Anthropic、Google、Azure、Ollama 和任何 OpenAI 兼容 API 提供统一界面
- **模型比较**，在单个对话中并排查看来自不同模型的响应
- **预设系统**，用于保存和快速切换模型配置和系统提示
- **工件**，用于内联渲染代码输出、文档和可视化
- **身份验证**，包括 LDAP、SSO 和社交登录支持
- **内置代码解释器**，用于支持的模型
- **提示缓存**，用于减少重复交互时的 API 成本
- **专注的范围**，在不使事情过度复杂化的情况下很好地处理聊天界面
- **活跃开发**，具有反应迅速的维护者和参与度高的社区
- **MIT 许可证**

---

## Open WebUI 的优势

- **超越聊天的平台**，包括笔记、频道、自动化、开放终端、语音/视频通话、图像生成和日历
- **知识与 RAG**，包含 9 个向量数据库、5 个提取引擎、混合搜索和重新排名以及智能体检索
- **Python 扩展性**，支持自定义工具、MCP 服务器、管道、OpenAPI 集成和社区市场
- **模型智能体**，为任何模型添加自定义说明、工具、知识和参数
- **企业功能**，包括 RBAC、SSO/OIDC/LDAP、SCIM 2.0、分析仪表板和评估竞技场
- **灵活的部署**，通过 Docker、Kubernetes、pip 或桌面应用，具有水平扩展和 OpenTelemetry

## 何时选择各个平台

**选择 LibreChat**（如果）您想要一个干净、专注的多提供商聊天界面，具有强大的模型比较功能。预设系统使在配置之间轻松切换成为可能，MIT 许可证为您提供最大的灵活性。

**选择 Open WebUI**（如果）您需要更广泛的平台，包括知识库、团队协作工具、Python 扩展性或企业功能，如 SCIM 和分析。

**同时运行两者。** 它们连接到相同的后端。一些团队使用 LibreChat 进行快速个人聊天，使用 Open WebUI 进行带有知识库和工具的协作工作。

## 一起使用它们

两个项目都连接到相同的后端（Ollama、OpenAI 等），因此您可以并排运行两者。一些团队使用 LibreChat 进行快速个人聊天，使用 Open WebUI 进行团队协作和知识工作。

## 常见问题

**LibreChat 和 Open WebUI 如何比较？**
LibreChat 在多提供商聊天界面方面表现非常好。Open WebUI 还包括知识库、团队协作、扩展性和企业功能。范围不同，两者都值得关注。

**LibreChat 是免费的吗？**
是的。LibreChat 采用 MIT 许可证，可以自由自托管。

**我可以同时使用 LibreChat 和 Open WebUI 吗？**
是的。两者都连接到相同的后端（Ollama、OpenAI 等），因此您可以并排运行两者。
