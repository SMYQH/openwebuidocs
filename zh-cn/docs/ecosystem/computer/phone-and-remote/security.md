# 安全模型

用一句话概括：**已登录用户对 Computer 能访问的所有内容拥有完全访问权限：文件系统和 shell，等同于 SSH 会话。** 这个范围有多大，由你在安装时决定：直接在主机上运行，是整个机器；[在 Docker 中运行](/ecosystem/computer/install/docker)，则精确到你挂载的文件夹。在该边界内没有每用户隔离。Computer 是为一个受信任的拥有者（或一小群你愿意给予 SSH 访问权限的人）构建的，以下所有内容都基于此。

这就是为什么[远程访问选项](./index.md)都归结为同一个问题：谁能到达登录页面？

## 认证方式

- **密码模式（默认）。** 首次运行时，`cptr` 打印一个一次性设置 URL（`/?token=...`），仅在尚无用户时有效；它创建的账户是管理员。密码至少 6 个字符，并以 bcrypt 哈希存储。登录受限于每个 IP 每分钟 5 次尝试。
- **会话。** 登录设置一个 JWT cookie（`cptr_session`），有效期 30 天，超过一半时在使用中续期。每个请求都需要认证；没有 localhost 绕过。
- **注册。** 管理员可以启用自行注册（**设置 → 管理**）。新注册用户获得 **pending** 角色，在管理员批准前无法登录。角色用于门控管理设置；它们不会在用户之间创建文件系统隔离。
- **其他模式。** `pam`（Linux 系统用户）和 `trusted_header`（反向代理 SSO）用于代理设置；参见[反向代理和 SSO](./reverse-proxy)。

## 使所有会话登出

登出仅删除该浏览器的 cookie，更改密码不会使现有会话失效。要一次性撤销所有会话：将 `config.toml` 中的 `[server] secret` 替换为新的随机值并重启 `cptr`。然后轮换其他任何授予访问权限的凭据：网关 API 密钥、消息机器人令牌。

## 无人值守路径以完全审批模式运行

网关 API、[消息机器人](/ecosystem/computer/automate/messaging-bots)和定时任务以完全工具审批模式运行：没有人为每个命令确认。任何能触发它们的人（允许的机器人发送者、持有网关密钥的客户端）都是在你的机器上以你的身份操作。像保护登录密码一样保护这些凭据。

## 审计日志

默认关闭。启用时记录变异的 API 请求（敏感数据经过编辑）。这对于审查发生了什么很有用，但它不是完整的终端转录。设置：[日志和健康](/ecosystem/computer/operate/logs-and-health)。

## 不应该做的事

- **不要将其直接暴露给公共互联网。** 仅有密码门控的公共 URL 是一个公开邀请；使用 [Tailscale](./tailscale) 或在[隧道前面放置提供商认证](./cloudflare-and-ngrok)。
- **不要与你不愿意给予 shell 访问权限的人共享实例。** 账户不是隔离。
- **不要在传递客户端标头的代理后面运行 `trusted_header` 模式。** 代理必须剥离并拥有身份标头；参见[反向代理和 SSO](./reverse-proxy)。

:::info 如果它已经被暴露
重新绑定到 `127.0.0.1`（或停止隧道），轮换 `[server] secret` 并重启，重置密码并轮换网关密钥和机器人令牌，然后如果你启用了审计日志，查看[审计日志](/ecosystem/computer/operate/logs-and-health)。
:::
