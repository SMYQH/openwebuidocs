# 定时任务

按计划运行一个提示词："每个工作日上午 9 点运行测试"、"每周一总结新文件"、"每小时检查部署"。每次运行都是你选择的工作区中的真实聊天，因此你可以精确读取发生了什么。

## 创建任务

最快的方式是直接要求。在任何聊天中，在工作应该发生的工作区中：

> 安排一个任务，每个工作日上午 9 点运行测试套件，并报告失败的测试及其错误输出。

代理有这个工具；批准工具调用，任务就存在了，工作区和模型已经从聊天中设置好。然后打开**定时任务**页面检查下次运行时间，并在相信计划之前点击**立即运行**一次。

定时任务页面也是手动创建和管理任务的地方：名称、提示词、模型、工作区，以及从频率构建器（每小时、每天、每周……）或原始 RRULE 生成的计划。计划是 iCalendar RRULE；原始字段接受 RRULE 支持的任何内容：

| 计划 | RRULE |
| --- | --- |
| 每天 09:00 | `RRULE:FREQ=DAILY;BYHOUR=9;BYMINUTE=0` |
| 每周一 | `RRULE:FREQ=WEEKLY;BYDAY=MO` |
| 每小时 | `RRULE:FREQ=HOURLY;INTERVAL=1` |
| 一次，然后不再执行 | `RRULE:FREQ=DAILY;COUNT=1` |

:::info 运行是无人值守的
定时运行以完全工具审批模式执行：没有人点击允许。给任务一个你熟悉的工作区和一个精确说明做什么（以及不做什么）的提示词。
:::

## 一次运行产生什么

每次运行创建：

- 所选工作区中的一个**真实聊天**，在侧边栏中可见，包含完整的任务活动，以及
- 任务上的一个**运行历史记录**，显示成功或错误（带错误文本）。

想要一个通知？添加一个通知目标一次，然后在任务的提示词中设置条件（"仅在失败时通知我"）：代理有一个通知工具，并自行决定何时使用它。每个完成/失败运行的全面事件通知也存在。参见[通知和 webhook](./notifications)。

## 使用 webhook 触发任务

任何定时任务也可以从外部启动（CI、cron 作业、其他服务），无需浏览器会话：

1. 在任务上，生成其 webhook URL。它包含一个秘密令牌。
2. `POST` 到该 URL 以立即运行任务。
3. 如果泄露，随时撤销或重新生成 URL。

如果 POST 有一个 JSON 体，提示词可以通过 `{{webhook_payload}}` 占位符使用它，因此 CI 失败负载、表单提交或警报可以直接流入任务的指令中。

## 一个好的第一个任务

名称：`Morning test check`。计划：`RRULE:FREQ=DAILY;BYHOUR=9;BYMINUTE=0`。提示词：

```text
在此工作区中运行测试套件。不要更改任何文件。
报告通过或失败；如果失败，包括失败的测试和错误输出。
```

点击**立即运行**，阅读它创建的聊天，一旦输出是你可以在几秒钟内判断的内容，就保持启用状态。
