# MCP 和 OpenAPI 工具服务器

通过在**设置 → 管理 → 工具服务器**中连接外部工具服务器，为代理提供内置工具之外的工具。支持三种服务器类型：

| 类型 | 你配置的内容 | 运行方式 |
| --- | --- | --- |
| **OpenAPI** | 规范 URL + 路径（默认 `openapi.json`） | Computer 调用 HTTP API；每个具有 `operationId` 的操作成为一个工具 |
| **MCP（远程）** | 服务器 URL | 流式 HTTP 传输到远程 MCP 服务器 |
| **MCP（stdio）** | 命令、参数、环境变量、工作目录 | Computer 在本地启动进程，保持其运行，如果崩溃则重新启动 |

MCP 支持需要可选依赖组（Docker 镜像已包含）：

```bash
pip install 'cptr[mcp]'
```

## 添加服务器

1. 在**设置 → 管理 → 工具服务器**中，添加一个服务器并选择其类型。
2. 给它一个 **id**：仅限小写字母、数字和下划线（`[a-z0-9_]`）。工具以 `<server_id>_<tool_name>` 的形式呈现给模型，因此保持简短且可识别。
3. 填写类型特定字段：OpenAPI 的规范 URL 和路径，远程 MCP 的 URL，或 stdio MCP 的命令/参数/环境变量/工作目录。
4. 如果服务器需要，添加认证：bearer 密钥和/或自定义标头。对于 stdio 服务器，通过进程环境传递凭据。
5. 点击**验证**：它会连接并列出发现的工具，以便你在保存前检查你启用的内容。

保存后，这些工具与内置的文件、终端和网络工具一起可供代理使用。

:::info
stdio MCP 服务器是与 Computer 本身在相同账户下运行在 Computer 主机上的真实进程，远程服务器会接收代理发送的任何参数。只连接你信任其工作区上下文的服务器。
:::
