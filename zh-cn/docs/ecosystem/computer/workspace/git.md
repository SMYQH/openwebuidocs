# Git

当工作区是一个 git 仓库时，git 状态条显示当前分支和更改的文件，每个文件的添加和删除行数。展开它以暂存、差异比对、提交、分支和推送：完整的日常 git 操作无需命令行，这在当你手中的设备是手机时尤为重要。

## 审查更改

打开任何更改的文件以查看其差异，使用并排或统一视图。单词级高亮标记行内更改的内容，你可以隐藏仅空白的更改以减少格式化文件中的噪音。图片和其他非文本文件显示清晰的更改/添加/删除状态，而不是无意义的行数。

精确暂存你想要的更改；暂存集是提交的内容。

## 提交和同步

编写提交消息，或接受从暂存更改生成的建议提交标题和描述，并在提交前编辑。状态条显示你领先或落后远程多少个提交，推送和拉取一键可达。

## 分支和隐藏

从分支选择器你可以创建、切换、重命名和删除分支。如果切换需要干净的工作树，Computer 会提供在切换时隐藏你的更改，隐藏视图让你以后检查和恢复隐藏。没有任何内容被静默丢弃。

## Worktrees

Worktree 选择器列出仓库的 git worktrees 并允许你创建新的。Worktree 是其自己的目录，有其自己的检出分支，所以打开它会作为其自己的工作区打开，有自己的标签页、终端和布局。这是同时处理两个分支的方式，而不是来回弹跳单个检出。参见[添加和切换工作区](./workspaces)了解工作区的行为方式。
