# 终端

打开一个终端标签页，你会获得一个完整的 PTY 支持的主机 shell，从工作区文件夹启动。它是你真实的 shell：你的 PATH、你的凭据、你的工具。任何你通过 SSH 运行的东西都在这里运行：测试、构建脚本、`htop`、vim、docker。

## 会话持久存在

关闭标签页、关闭浏览器、将手机放进口袋；shell 及其正在运行的内容在主机上继续运行。从任何设备重新连接，会话仍然存在，滚动历史会回放，所以你可以在办公桌开始一个长时间构建，然后在火车上检查输出。

打开任意数量的终端，以标签页或并排分割方式：一个开发服务器、一个日志、一个用于探查的 shell。布局随工作区布局保存。

## 终端编码代理

任何 CLI 工具都能工作，包括终端编码代理。在终端标签页中运行 Gemini CLI 或任何其他代理，就像在本地一样，它像每个其他会话一样在断开连接后持续存在。如果你更希望 Claude Code、Codex 等出现在聊天模型选择器中，将它们连接为[原生编码代理后端](/ecosystem/computer/ai/coding-agents)。

## 什么不会持续

会话存在于 `cptr` 进程内部。它们在以下情况下结束：

- `cptr` 进程本身停止或重启
- 主机重启，或进入睡眠并挂起其进程

如果长时间运行的任务必须超越这些情况，参见[保持运行](/ecosystem/computer/phone-and-remote/keep-it-running)以防止主机睡眠，或在 `tmux`/`nohup` 下运行任务，使其完全脱离会话。
