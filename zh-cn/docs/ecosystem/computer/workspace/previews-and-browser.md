# 预览本地应用和浏览

两种方式让网页出现在你的工作旁边：**端口预览**打开主机上正在运行的开发服务器，**浏览器标签页**打开任何 URL。两者都位于与你的文件和终端相同的标签页和分割布局中。

## 预览本地开发服务器

按常规方式在工作区终端中启动你的应用：

```bash
npm run dev
```

当 Computer 检测到进程在监听端口时，该端口出现在文件浏览器的**端口**行中。点击 `:port` 在预览标签页中打开应用，就在产生其日志的终端旁边。

预览连接到实时进程。编辑文件，让开发服务器重新加载，预览显示结果。如果进程退出（崩溃、主机重启、`cptr` 重启），端口消失；重新启动服务器并重新打开它。

### 在真实设备上测试

因为预览可以从任何登录到你的实例的设备工作，这是在真实硬件上测试的简单方法：开发服务器在你办公桌的机器上运行，你在实际的手机或平板上打开其端口预览，以检查触摸目标、视口行为和布局。无需部署，无需将项目复制到任何地方。在桌面上保持终端在分割中打开，以便在你在手机上点击时观察日志。

端口预览是为登录到你的实例的人准备的；它不是公共 URL，也不会为它服务的应用添加认证。

## 浏览任何 URL

添加浏览器标签页以访问任何网站（暂存站点、API 文档、问题跟踪器），无需离开工作区。后退、前进和重新加载按预期工作，标签页像其他所有内容一样位于你的布局中。

浏览器会话是实时会话，而不是持久的配置文件：`cptr` 重启后，你可能需要重新加载页面并再次登录网站。

这些标签页背后的浏览器可以是托管浏览器或个人 Chrome，取决于实例在设置中的配置方式。AI 的网络浏览工具使用相同的来源。

:::warning
已登录的浏览器会话是这些账户的真实访问权限：任何可以使用你的 Computer 实例的人都可以在你登录的任何网站上以你的身份操作。保持实例私有，不要在对其他人可访问的实例上保持登录敏感账户。
:::
