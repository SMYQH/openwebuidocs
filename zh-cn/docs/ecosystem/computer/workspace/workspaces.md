# 添加和切换工作区

打开侧边栏中的工作区选择器，通过绝对路径添加主机上的任何文件夹，例如 `/Users/you/src/project`。你也可以在浏览时直接从选择器创建新文件夹，因此启动一个全新项目不需要终端。

任何文件夹都可以。它不需要是 git 仓库；一个笔记、PDF 或下载的目录就是一个完全有效的工作区。如果它是一个仓库，git 面板会自动亮起。

## 工作区的范围

选择工作区会将整个 UI 指向该文件夹：

- 文件浏览器和编辑器打开其树
- git 显示该检出的分支和更改
- 新终端在该目录中启动
- 新聊天在该项目的上下文中运行，并存储在项目内部
- 快速搜索保持在其文件、聊天和消息范围内

一个实例，多个工作区：从选择器切换项目，每个工作区保持其自己的打开标签页、分割和布局。切换走再回来，一切都在你离开的地方，在任何设备上。

## 路径是主机路径

你添加的路径是运行 `cptr` 的机器上的路径，而不是你浏览用的手机或笔记本电脑。

在 Docker 中，这意味着**容器内部**的路径。先挂载你的项目，然后将容器端的路径添加为工作区：

```bash
docker run --rm -it \
  -p 8000:8000 \
  -v cptr-data:/data \
  -v "$PWD:/workspace" \
  -w /workspace \
  ghcr.io/open-webui/computer:latest
```

这里的工作区路径是 `/workspace`。如果某个文件夹没有出现在选择器中，通常是因为它没有挂载到容器中。

## 单独的检出，单独的工作区

将仓库的每个检出放在其自己的工作区中。git worktree 是其自己的目录，有其自己的检出分支，所以将其添加为单独的工作区。[git 面板中的 worktree 选择器](./git#worktrees)在你创建或打开一个时为你完成此操作。这样两个分支可以被检出并并排工作，而不会争夺一个工作树。

## 权限

`cptr` 以启动它的操作系统账户运行。工作区的可读和可写权限与该账户的权限完全一致：只读挂载和文件所有权照常适用。将一个文件夹添加为工作区不会授予或限制任何内容；它只是告诉 UI 在哪里查找。
