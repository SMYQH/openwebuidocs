# 搜索和聊天

按下 `Cmd+K`（Windows/Linux 上为 `Ctrl+K`）进行搜索。当工作区激活时，结果限制在该工作区内：文件名、聊天以及其中的消息。`Cmd+Shift+F` 打开更广泛的搜索。查询为空时，对话框显示最近的项目，`Cmd+1` 到 `Cmd+9` 直接跳转到其中一个。

这意味着"AI 在哪里解释那个迁移的？"一次搜索即可找到，从任何设备；聊天历史与文件一起被索引。

## 聊天是工作区中的文件

聊天属于项目记录。每个存储为 `<workspace>/.cptr/chats/<chat_id>.json` 的文件，位于项目文件夹本身。因为它们是普通文件，所以：

- **可搜索**：通过与其他所有内容相同的搜索找到
- **可移植**：复制文件夹，聊天历史也随之而来
- **可提交**：如果你希望决策与代码一起版本化，将 `.cptr/chats/` 检查到 git 中（如果你不希望，则添加到 `.gitignore`）

## 管理聊天

- **在任何地方继续。** 从侧边栏或搜索打开任何聊天并继续它：同一个聊天，任何设备。
- **分支而非偏离。** 从任何回复分支聊天，以在新副本中尝试不同方向，同时保持原始内容完整。
- **从侧边栏删除。** 直接从最近聊天列表中删除你不再需要的聊天。
