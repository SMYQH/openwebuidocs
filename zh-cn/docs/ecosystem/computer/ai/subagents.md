# 子代理

AI 可以使用 `delegate_task` 工具将工作委派给并行的子代理。每个子代理是一个完整的聊天，具有工具访问权限（可以读取文件、运行命令和搜索），其结果在完成后被注入回父对话。你会在聊天中看到委派作为工具活动；每个工作者的输出都是可审查的。

## 设置

在**设置 → 管理 → 子代理**中配置：

| 设置 | 默认值 |
| --- | --- |
| 启用 | 开 |
| 后台子代理 | 开 |
| 最大并发数 | 20 |
| 每个子代理的最大迭代次数 | 30 |
| 最大输出字符数 | 30000 |
| 自定义系统提示 | 无 |

后台子代理在父聊天继续进行的同时保持工作，并在完成时注入其结果。它们与进程绑定：服务器重启会中断它们，而不会在之后恢复。

## 何时使用

子代理在任务可以拆分为**独立的**问题时表现出色（找出失败的测试、列出调用者、扫描部署日志），每个问题都有自己的答案，且没有需要编辑的共享文件。父任务得到三个简洁的结果，而不是一次冗长的串行调查。

当步骤相互依赖或涉及相同的文件时跳过它们：并行工作者进行的顺序编辑会产生冲突，而不是速度。一个受监督的代理更容易审查。更多工作者也会增加外部请求的数量，而且它们不会增加隔离：每个子代理与父代理拥有相同的机器访问权限。
