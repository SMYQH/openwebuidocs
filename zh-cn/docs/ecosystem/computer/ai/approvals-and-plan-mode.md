# 审批、计划模式和审查

每个聊天都有一个工具审批模式，从聊天输入中的 **+** 菜单设置：

| 模式 | 无需询问即可运行 | 仍然会询问 |
| --- | --- | --- |
| **ask** | 无 | 每次工具调用 |
| **auto** | 只读工具：`read_file`、`list_directory`、`search_files`、`web_search`、`read_url`、`search_chats`…… | 写入和命令：`write_file`、`edit_file`、`run_command`、`send_input`、`create_automation`…… |
| **full** | 所有 | 无 |

当调用需要审批时，聊天中的工具卡片会显示内联的**允许**/**拒绝**按钮。**auto** 是日常使用的设置：AI 可以自由探索，但未经你同意，不会更改磁盘上的任何内容，也不会运行任何命令。

## 原生代理如何映射模式

编码代理后端将聊天模式转换为其自己的权限系统：

- **Claude Code：** full → `bypassPermissions`，auto → `acceptEdits`，ask → `default`
- **Codex：** 映射到其审批策略和沙箱
- **Cursor、Grok、Cline：** 显示（或自动应答）它们自己的权限请求，这些请求在聊天中如同 Computer 的一样出现

## 计划模式

从 **+** 菜单切换或输入 `/plan`。计划模式移除写入工具：代理以只读方式进行研究，生成计划（它可以创建计划制品），只有在您批准后才开始实施。对于任何不熟悉或高影响的工作，先使用计划模式：你可以在单个文件更改之前看到方案。

## 输出编辑

你可以审查和编辑 AI 生成的输出，然后再应用：修复路径或修剪更改，而无需重新提示。

## 审查更改了什么

监督主要是阅读差异。在任务完成之后（或期间）：

1. 打开 [git 面板](/ecosystem/computer/workspace/git)并阅读更改的路径和差异。代理在真实的仓库中工作，所以面板会精确显示它做了什么。
2. 如果更改或建议的命令不清楚，在同一个聊天中询问："先不要创建迁移。先给我展示受影响的调用者。"代理会保持其上下文。
3. 只批准你理解的内容。拒绝破坏性命令，并要求先解释或进行空运行。
4. 当差异与你要求的内容匹配时提交；有针对性的测试运行比相信摘要更便宜。

这在手机上也是一样的：工具卡片、任务进度和 git 面板都在同一个工作区视图中。

:::info
无人值守路径（网关 API、消息机器人和定时任务）始终在 **full** 审批模式下运行。只将它们连接到你在没有提示的情况下也能接受的工作。参见[定时任务](/ecosystem/computer/automate/scheduled-tasks)。
:::
