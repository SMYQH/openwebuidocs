# 委派整个工作，回来时拿到交付物

你的团队需要为团建活动选择一个城市：20 人，三个候选城市，航班、场地、签证问题、预算上限。那是一个下午的标签页和电子表格。或者是一个精心编写的简要说明，在 9:00 交给代理，当你午餐回来时，一个包含报告、比较表格和来源的文件夹在等着你。

这里的技巧不是提示；而是*委派*：明确的目标、定义好的交付物，以及你控制的检查循环。

**你需要：** 已连接[模型](/ecosystem/computer/ai/connect-a-model)。网络搜索在零配置下工作（DuckDuckGo）；搜索提供商密钥可以改善结果，启用[浏览器](/ecosystem/computer/ai/web-search-and-browsing)让它读取搜索片段无法涵盖的页面。

## 操作步骤

1. **给工作一个家。** 新文件夹 `offsite-research/`，作为工作区打开。交付物将是这里的文件，而不是聊天中的滚动记录。

2. **批准计划，而不是每次按键。** 打开[计划模式](/ecosystem/computer/ai/approvals-and-plan-mode)，像对人一样写简要说明：

   > 比较里斯本、布拉格和瓦伦西亚，用于 20 人、3 晚的十月团建，预算 €40k。我需要：report.md 包含推荐和理由；costs.csv 比较从柏林/伦敦/纽约的航班、酒店和每个城市的场地估算；sources.md 包含每个数字的链接。标记与美国和印度护照持有者相关的签证问题。

   它会带着计划回来：它将搜索什么，如何分配工作，每个文件中放什么。调整（"使用周中航班价格"），然后批准。

3. **让它分派工作。** 对于这样的任务，代理可以启动[子代理](/ecosystem/computer/ai/subagents)：每个城市一个进行并行研究，结果汇总回主任务。你会在工具日志中看到委派发生。

4. **走开；按你的方式检查。** 任务在服务器端运行。在手机上瞄一眼：聊天显示它当前在做什么，如果它遇到了真正模糊的问题，你的回复可以引导它，而无需重新启动任何内容。

5. **回来时看到文件，审计重要的那些。** `report.md`、`costs.csv`、`sources.md`，像工作产品一样坐在文件浏览器中，因为它们就是。聊天是完整的回放：每次搜索、它读取的每个页面、每个数字的来源。像检查实习生的工作一样抽查：挑选决定所依赖的两个数字，追踪到它们的来源。

6. **像经理一样迭代，而不是提示者。** "瓦伦西亚的场地估算看起来偏低；根据实际报价重新检查，并添加一行降雨概率"是一条后续消息，而不是一个新项目。相同的工作区，相同的上下文，更新的文件。

## 为什么这样有效

交付物优先的简要说明改变了一切：代理优化为完成文件，而不是保持对话，你得到可以比较、分享和重新生成的制品。前面的计划模式意味着你在判断成本低的时候（在工作之前）使用你的判断力，而可回放的工具日志意味着信任是被验证的，而不是被假设的。同样的方法适用于市场扫描、竞争对手分析、文献综述和"将这文件夹的 CSV 变成简报"。
