# 在真实手机上测试你的开发服务器

结账流程在 Chrome 的设备模拟器中看起来完美。上次在真实的 iPhone 键盘覆盖了提交按钮之前，它看起来也很完美。模拟器没有真实键盘、真实安全区域或真实触摸物理学；你的手机有，而你的开发服务器已经在运行。

**你需要：** 完成快速入门，并且你的手机可以访问 Computer（同一 Wi-Fi 就足够了：[手机访问](/ecosystem/computer/phone-and-remote/)；如果你希望从任何地方都能工作，使用 Tailscale）。

## 操作步骤

1. **在它已经存在的地方启动应用。** 在项目工作区中，打开一个终端并像往常一样运行开发服务器：

   ```bash
   npm run dev
   ```

2. **找到端口。** 当工作区终端中的进程开始监听时，端口会出现在文件浏览器上方的**端口**行中。点击 `:5173`（或任何端口）打开一个预览标签页。预览代理了真实的本地进程；无需部署任何内容。

3. **在手机上打开它。** 从手机登录 Computer 并打开相同的预览标签页。你现在在真实的触摸屏上操控你办公桌上的开发服务器。

4. **修复和检查循环。** 当你编辑时，在手机上保持预览打开（或从手机编辑；这是同一个工作区）。热重载正常工作，因为是同一个服务器；预览旁边的终端标签页在你点击流程时显示请求日志和错误。

5. **记录你的发现。** 当键盘再次覆盖按钮时，在会有人看到的地方记录下来：工作区中的笔记，或通过 `@` 提及文件的聊天消息，这样 AI 在你要求它修复 CSS 时就拥有了上下文。

## 为什么这样有效

预览通过 Computer 自己的认证会话到达实时进程，所以没有任何内容暴露给你的网络，没有隧道、二维码或部署步骤。当服务器死亡或主机重启时，预览也会消失；重新启动进程并重新打开端口。它反映现实，这正是全部意义所在。
