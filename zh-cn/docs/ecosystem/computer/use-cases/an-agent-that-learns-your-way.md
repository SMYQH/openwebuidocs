# 一个学习你工作方式的代理

第三次向 AI 解释你的发布流程时，应该有所改变。不是模型；而是设置。聊天机器人和同事的区别在于，同事不需要听两次说明：他们保留检查清单、记住你的偏好，并且能够找出"我们六月决定的那件事"而无需你重新讲述。Computer 拥有所有这三种机制；本指南将打开它们。

**你需要：** 一个已连接的[模型或编码代理](/ecosystem/computer/ai/)。仅此而已。

## 操作步骤

1. **将第三次解释变成技能。** 你刚刚再次带着 AI 走了一遍发布流程：升级版本、从合并的 PR 生成变更日志、标签格式、烟雾测试、公告发布。这次，最后说：

   > 将整个过程保存为一个名为 release-checklist 的技能。包括标签格式和公告模板。

   代理将一个 `SKILL.md` 写入工作区的 `.cptr/skills/` 文件夹。它是一个普通文件；打开它，编辑措辞，如果你愿意，可以随仓库一起提交。下次发布时，输入 `$release-checklist`，整个方法就会加载。如果你已经为 Claude Code 或 Codex 保留了技能，Computer 也找到了它们；它会读取 `.claude/skills`、`.codex/skills`、`.agents/skills` 及其全局的 `~/` 等效目录。

2. **让技能自我优化。** 在管理员技能页面上启用后台技能审查。每隔一段时间（默认：每 10 次轮次），Computer 会查看技能实际使用情况并提出更新：你在发布过程中修正的边界情况会被纳入检查清单，而不是只存在于你的脑海中。你审查建议的差异；不会静默更改任何内容。

3. **为个人化内容开启记忆。** 技能保存流程；记忆（设置）保存*你*："偏好 squash 合并"、"周五绝不部署"、"时区是 CET"。它会从你的修正中积累，并自动在未来聊天中回忆。它所持有的所有内容都是可检查和可删除的，这正是它成为功能而非负担的原因。

4. **使用你一直在生成的记录。** 每个聊天都是工作区记录的一部分，代理可以搜索它。所以问你问同事的问题：

   > 搜索我们过去的聊天：六月关于重试退避我们做了什么决定，为什么？

   它会提取实际决策及其上下文。跨设备，几个月后，仍然存在，因为聊天是工作区 `.cptr/` 文件夹中的文件。

5. **推广值得传播的内容。** 在一个仓库中证明了自己的技能可以移动到 `~/.cptr/skills`，每个工作区都可以使用。你的全局技能库加上你的记忆配置，实际上就是一个根据你的工作方式训练过的代理，可跨每个项目和你插入的每个模型移植。

## 为什么这样有效

每种机制关闭一个不同的循环：技能阻止你重新教授*流程*，记忆阻止你重复说明*偏好*，聊天搜索阻止你重新争论*决策*。这些都不是魔法；它们是你可读、可编辑和可删除的文件和设置，这正是它们能够积累而不是偏离的原因。两个月后，你拥有的助手明显比你安装时更好，而这种改进的每一部分都坐在你拥有的一个文件夹中。
