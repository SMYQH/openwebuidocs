# 一个主动给你发消息的助手

星期二早上 7:45。你的手机震动，收到你自己电脑发来的 Telegram 消息：

> 早上好。中午前有雨。新闻订阅中有三条与你关注列表匹配的内容；有趣的是 SDK 废弃通知，总结在工作区中。你的副项目测试夜间通过了。10 点会议前没有任何需要你处理的事情。

你没有要求。这就是重点。聊天机器人等待；助手主动出现。本指南将 Computer 变成那种主动出现的类型。

**你需要：** 已连接[模型](/ecosystem/computer/ai/connect-a-model)、已设置 [Telegram 机器人](/ecosystem/computer/use-cases/message-your-computer)，以及主机[早上保持唤醒](/ecosystem/computer/phone-and-remote/keep-it-running)。网络搜索开箱即用（DuckDuckGo 不需要密钥）。

## 操作步骤

1. **给它一个总部。** 创建一个像 `assistant/` 这样的文件夹，将其作为工作区打开，并在其根目录下放置 `.cptr/system.md` 文件，给助手常设指令。该文件成为该工作区中每个聊天的系统提示：

   > 你是我的个人助理。要简洁具体。我关心：Rust 生态系统、我们的开源仓库、柏林天气。从不编造事实；当什么都没有时说"没有找到"。每次简报以"没有什么需要你处理"或确实需要处理的一件事结尾。

2. **开启记忆。** 在设置中启用记忆。现在修正会固定下来：告诉它一次你不关心加密货币新闻，未来的简报就知道了。你可以查看和删除它记住的任何内容。

3. **给它联系你的渠道。** 在**设置 → 通知**中，添加一个指向你的 Telegram 机器人的**机器人**目标并测试一次。这是助手有话要说时使用的渠道。

4. **让它安排自己的简报。** 在其工作区聊天中告诉助手：

   > 每天早上 7:40，编译我的早间简报：今天的柏林天气、匹配我兴趣的新闻头条（搜索网络），以及昨晚 ~/code/sideproject 中测试运行的结果。将完整简报写入 briefs/today.md，然后将 5 行摘要作为通知发送给我。将其设置为定时任务。

   它自己创建任务；批准工具调用，快速浏览**定时任务**页面查看下次运行时间，点击**立即运行**一次进行预演。"发送给我"部分是代理自己的通知工具进行传递，这意味着助手决定什么值得说，并且可以在没有什么需要时保持沉默："只有需要我注意时才通知我"是一个有效的常设指令。

5. **回复它。** 这是一个真实的聊天，所以回复有效："展开说一下 SDK 的事情"会以相同的上下文从同一部手机继续相同的对话。

6. **升级：事件驱动的通知。** 定时不是唯一的触发器。每个定时任务可以暴露一个秘密 webhook URL，调用者的 JSON 可以通过 `{{webhook_payload}}` 提供给提示词。将你的 uptime 监控或 CI 指向一个：

   > 负载描述了一个失败。调查：检查此工作区中的服务日志，用 3 行总结可能的原因。

   现在助手不仅在 7:40 给你简报；它在凌晨 3 点进行调查，并通过短信给你诊断结果，而不仅仅是警报。

## 为什么这样有效

主动性就是"定时加传递路径"，两个部分都是可见的：每次简报都是一个真实的聊天，带有完整的工具日志，所以当它说测试通过了，你可以打开运行并看到命令。个性文件和记忆是让它感觉像*你的*而不是一个 cron 作业的原因；你写入 `.cptr/system.md` 的诚实规则是保持每日无人值守摘要可信赖的关键。从一个你可以验证的简报开始，只有当它赢得了你的信任时才扩大授权范围。
