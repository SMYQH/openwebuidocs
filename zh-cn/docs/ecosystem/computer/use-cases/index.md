# 人们实际用它做什么

十三个真实工作流，以你今天就可以执行的步骤编写，从"我从未打开过终端"到"我正在并行运行三个代理"。每个都说明需要先设置什么，以及成功运行时你会看到什么。找到你的层级，然后从此开始。

## 从这里开始（无需编码）

如果 Computer 是你的第一个此类工具，这三个需要零终端和零代码。安装就是复制粘贴两个命令，之后的一切都是点击和用自然语言输入。

| 场景 | 操作指南 |
| --- | --- |
| 你的下载文件夹有 400 个文件，混乱不堪，今天就是整理的日子。 | [用 AI 清理杂乱的文件夹](./clean-up-a-messy-folder) |
| 一个学校或研究项目，散落在两台设备上的 PDF、笔记和语音备忘录中。 | [跟随你的项目文件夹](./a-folder-that-follows-you) |
| 你想要一个俱乐部或作品集的网站，并且你想真正拥有它。 | [在 AI 的陪伴下构建你的第一个网站](./first-website) |

## 你的电脑，在你的口袋里

你用电脑做真正的工作，并希望离开办公桌时也能带着它。这些添加了远程访问，并温和地开始接触终端和 git。

| 场景 | 操作指南 |
| --- | --- |
| 你想从聊天应用而不是浏览器中问你的电脑"构建通过了吗？" | [从 Telegram 给你的电脑发消息](./message-your-computer) |
| 你在火车上时，一个 bug 报告来了。分支在你办公桌上的机器上。 | [从手机推送修复](./fix-from-your-phone) |
| 布局在 devtools 的手机模拟器中看起来不错。你以前被这个坑过。 | [在真实手机上测试你的开发服务器](./test-on-a-real-device) |

## 你自己的个人助手

代理之所以成为热门话题的原因：一个能自主行动、完成整个工作，并且为你工作越久越好的代理。全部在你的机器上，使用你的密钥，每个工具调用都可审查。

| 场景 | 操作指南 |
| --- | --- |
| 早上 7:45，你的手机震动：是你的电脑，带着早间简报。你并没有要求。 | [一个主动给你发消息的助手](./an-assistant-that-texts-first) |
| 交给它"比较三个城市的团队活动地点"，回来时得到一份报告、一份电子表格和来源。 | [委派整个工作，回来时拿到交付物](./delegate-a-whole-job) |
| 第三次向 AI 解释你的发布流程时，它应该不需要你再重复了。 | [一个学习你工作方式的代理](./an-agent-that-learns-your-way) |

## 在真实代码上使用代理

适合将 AI 用于严肃项目的开发者：监督、自动化和编排。

| 场景 | 操作指南 |
| --- | --- |
| Claude Code 在家中进行重构，遇到了一个它不应单独做出的决定。 | [从任何地方监督编码代理](./supervise-a-coding-agent) |
| 测试套件应该每天早上运行，只在需要时通知你。 | [向你报告的早间检查](./scheduled-morning-checks) |
